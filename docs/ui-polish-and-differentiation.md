# UI改善・差別化フェーズ 実施記録

実施日: 2026-07-08
ブランチ: `experiment/ui-polish-and-differentiation`（mainから分岐した実験用ブランチ。mainには未マージ・未push。**一言で破棄可能**）
実施体制: fable-advisorサブエージェントによる実装（5コミット）＋オーケストレーターによる引き継ぎ仕上げ（boxShadow移行のコミット）と全機能のブラウザ実機検証

## 背景

Issue #2〜#18の機能実装・検証完了後、プロダクトオーナーから直接以下の指示を受けた:

> 名前の検索とか、分類とかもっとやりやすくしてほしい。同姓がいたりするはずだから探しやすくしたり、営業コーチをチャットボットにしたり。使いやすくするための工夫は既存アプリのシステムを参考に。UIをもっと美しく、他の似たアプリとの差別化要素も追加。ただし一言で戻せるように。

「一言で戻せるように」の要件を満たすため、全変更をmainから分岐した実験用ブランチに隔離した。破棄する場合は `git branch -D experiment/ui-polish-and-differentiation` の一言で消え、mainには一切影響しない。

## 実施内容（コミット順）

### 1. `c1a3079` — 会社・役職・紹介元のデータマッピング

- Supabaseスキーマに元々存在していた `contacts.company` / `contacts.role` / `contacts.introduced_by` カラムを `Person` 型・`personStorage.ts` のマッピングに追加。
- 人物詳細画面に会社名・役職の入力欄と保存ボタンを追加。
- Issue #17検証で「未マッピング」と指摘されていた同姓同名判別の重要情報源を解消。

### 2. `21f dc6a` — 共通ContactPickerModalの新設と全画面統一（CLAUDE.md 5.2準拠）

- `src/components/ContactPickerModal.tsx` を新設し、人物選択UIを1箇所に集約。
- 検索対象: 名前・会社・役職・メモ（プレースホルダー「名前・会社・役職・メモで検索」）。
- 各候補にアバター（頭文字）・会社/役職・分類・次回連絡日・次アクション・メモ抜粋を表示し、同姓同名を判別可能に。
- これまで検索がなかった営業コーチ（CLAUDE.md 5.2違反としてIssue #17で指摘）にもこのピッカーを配線。予定前ナビ・文面確認の既存ピッカーも共通コンポーネントに置き換え。

### 3. `2812494` — 営業コーチのチャットボット化（CLAUDE.md 5.7準拠）

- 一問一答（送信ごとに前の回答が消える）形式から、履歴が積み重なるチャットUIへ全面刷新。
- 選択した人物の過去の`coach_logs`をロードし、吹き出し形式（ユーザー右寄せ・AI左寄せ・タイムスタンプ付き）で表示。
- 人物を切り替えるとその人物の会話履歴に切り替わる。
- AI回答内の「次の行動」をハイライト表示。
- 送信中のローディング表示。
- 空状態（履歴なし）にはガイド文言とサンプル相談ボタン。
- CLAUDE.md 5.7の「Contact/SalesRoute/Taskを自動更新しない」原則は維持。

### 4. `cf37427` — 紹介チェーン表示（差別化要素）

- 人物詳細に「紹介チェーン」セクションを新設。
- 紹介元（この人を紹介してくれた人）の表示・設定（ContactPickerModalを再利用）。
- 「この人から紹介された人」の一覧表示（タップでその人物の詳細へ遷移）。
- 汎用CRMにはない「紹介のつながりを資産として可視化する」というこのアプリの核心コンセプトをUI化したもの。

### 5. `5fb50d2` — 全体的なUI磨き込み

- カードのシャドウ・角丸の一貫性向上。
- 人脈カード一覧に会社・役職の行を追加（一覧レベルでも同姓同名を判別可能に）。
- 人物重複排除ロジックの安全化。

### 6. `e1c4878` — 非推奨shadow*プロパティのboxShadow移行

- React Native Webで非推奨警告が出ていた `shadowColor`/`shadowOffset`/`shadowOpacity`/`shadowRadius`/`elevation` を、モダンな `boxShadow` スタイル文字列に置換（`PersonCard`/`Section`/`SectionCard`）。
- fable-advisorのセッション上限中断時に未コミットだった作業を、オーケストレーターがレビュー・型チェックの上で仕上げてコミット。
