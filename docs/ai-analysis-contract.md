# AI分析の入出力契約

このメモは、あとで本物のAI APIを接続するときのための型・出力方針です。

## 基本思想

営業マンが入力するのは、分類や温度感そのものではなく、会話で得た素材です。
AIはその素材から営業データを抽出し、人脈カード更新案を作ります。

## 後メモAIの入力

`AfterMemoAiInput`

- 対象人物
- 人脈カードの現在情報
- 予定前ナビで決めた目的
- 予定前ナビで決めた質問
- 質問への回答
- 話した内容
- 得た情報を全部貼ったメモ
- 営業マン本人が思う次アクション
- LINE履歴や送信予定文

## AIが抽出する営業データ

`ExtractedSalesData`

- 会話要約
- 相手の状況
- 課題
- 課題が放置された場合の影響
- 相手が望む状態
- 決裁者
- 関係者
- 予算感
- 期限
- 紹介可能性
- こちらから紹介できそうな人
- 紹介してほしい人
- 断り理由
- 信頼シグナル
- リスク
- 聞き漏れ

## AIが判断する営業状態

`QualificationAnalysis`

- 温度感
- 関係ステージ
- 営業ルート種別
- 顧客可能性
- 紹介元可能性
- 紹介先可能性
- 情報源価値
- 将来候補度
- 紹介依頼してよいか
- AIの確信度

## AIが作る人脈カード更新案

`PersonCardUpdateProposal`

- 分類
- ゴール
- 次アクション
- 次に聞く質問
- 次回連絡日
- 注意点
- 各スコア
- 追記するメモ

## AIフィードバック形式

`SalesCoachFeedback`

1. 結論
2. 理由
3. 科学的根拠
4. 営業現場への翻訳
5. 次の行動

## 最終出力

`AfterMemoAiOutput`

- 抽出した営業データ
- 営業状態の判定
- 人脈カード更新案
- 営業フィードバック
- 次アクション一覧
- LINE文案
- 通知設定案
- AI判断の根拠・仮定・不足情報
